## 💡 使用建議

### 選擇模擬模式

| 開發目標 | 推薦模式 | 理由 |
|---------|---------|------|
| 物體檢測 | Objects | 包含明確的物體邊界 |
| 深度處理 | Objects | 真實的深度變化 |
| 相機標定 | Pattern | 規則的幾何圖樣 |
| 演算法壓力測試 | Random | 隨機資料測試穩健性 |
| 完整流程開發 | Objects | 最接近真實場景 |

### 開發流程建議

1. **初期開發**: 使用 `Objects` 模式建立基本流程
2. **演算法測試**: 使用 `Random` 模式測試邊界情況
3. **幾何校正**: 使用 `Pattern` 模式驗證對齊與標定
4. **整合測試**: 切換不同模式確保通用性

## 📐 技術規格

### RGB 影像
- 解析度: 1920×1080 (Full HD)
- 色彩空間: BGR (OpenCV 標準)
- 位元深度: 8-bit per channel
- 格式: uint8 numpy array

### 深度影像
- 解析度: 1280×800
- 單位: 毫米 (mm)
- 範圍: 0-10000mm (有效範圍約 500-2000mm)
- 位元深度: 32-bit float
- 格式: float32 numpy array

### 內參 (模擬)

**RGB 相機:**
- fx: 1081.37 像素
- fy: 1081.37 像素
- cx: 960.0 像素 (影像中心)
- cy: 540.0 像素 (影像中心)

**深度相機:**
- fx: 720.91 像素
- fy: 720.91 像素
- cx: 640.0 像素 (影像中心)
- cy: 400.0 像素 (影像中心)
